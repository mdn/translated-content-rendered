Un objeto inmutable es aquel cuyo contenido no se puede cambiar.Un objeto puede ser inmutable por varias razones, por ejemplo:

  • Para mejorar el rendimiento (al no haber planificados cambios futuros del objeto)
  • Para reducir el uso de memoria (haciendo referencias al objeto en vez de clonar el objeto entero)
  • Para la seguridad de los subprocesos (varios subprocesos pueden hacer referencia al mismo objeto sin interferir entre sí)
