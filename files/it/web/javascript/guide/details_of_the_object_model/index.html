JavaScript un linguaggio ad oggetti basato su prototipi, piuttosto che sulle classi. A causa di questa diversa base, pu essere meno evidente come JavaScript permette di creare gerarchie di oggetti e di avere l'ereditariet delle propriet e dei loro valori. Questo articolo cerca di chiarire questo aspetto.

Questo capitolo presuppone una certa familiarit con JavaScript e con l'uso delle funzioni per la creazione di semplici oggetti.

Linguaggi class-based vs. linguaggi prototype-based

I linguaggi ad oggetti basati su classi, come Java e C++, si basano su due entit distinte: le classi (classes) e le istanze (instances).

  • Una classe definisce tutte le propriet (considering methods and fields in Java, or members in C++, to be properties) che caratterizzano una determinata collezione di oggetti. Una classe un'entit astratta, pi che un membro del gruppo di oggetti che descrive. Per esempio, la classe Employee pu rappresentare il gruppo di tutti i dipendenti.
  • Un'istanza, d'altra parte, l'istanziazione di una classe; cio uno dei suoi membri. Per esempio Victoria pu essere un'istanza della classe Employee, che rappresenta un particolare individuo come un dipendente. Un'istanza ha esattamente le stesse propriet della classe a cui appartiene (n pi, n meno).

Un linguaggio basato su prototipi, come JavaScript, non fa questa distinzione: ha solo oggetti. Introduce la nozione di oggetto prototipo (prototypical object), un oggetto usato come modello da cui prendere le propriet iniziali per un nuovo oggetto. Ogni oggetto pu specificare le sue propriet, anche quando viene creato o in fase di esecuzione.Inoltre, ogni oggetto pu essere associato ad un altro oggetto come prototipo, consentendo al secondo oggetto di condividere le propriet del primo.

Definizione di una classe

Nei linguaggi basati su classi, le classi vengono definite in class definition separate. In queste definizioni possibile specificare metodi speciali, chiamari costruttori (constructors), per creare istanze della classe. Un costruttore pu specificare i valori iniziali per le propriet dell'istanza ed eseguire altre elaborazioni adatte al momento della creazione. Per creare le istanze di una classe si utilizza l'operatore new associato al metodo costruttore.

JavaScript segue un modello simile, ma non prevede la definizione della classe separata dal costruttore. Invece, per creare oggetti con un particolare set di propriet e valori si definisce una funzione costruttore. Ogni funzione JavaScript pu essere usata come costruttore.Per creare un nuovo oggetto si utilizza l'operatore new associato a una funzione costruttore.

Sottoclassi e ereditariet

In un linguaggio basato su classi, si crea una gerarchia tra le classi attraverso le definizioni delle classi stesse. All'interno della definizione di una classe possibile specificare che la nuova classe una sottoclasse (subclass) di una classe esistente. La sottoclasse eredita tutte le propriet della superclasse e pu inoltre aggiungere nuove propriet o modificare quelle ereditate. Per esempio, assumiamo che la classe Employee include solo le propriet name e dept, e Manager una sottoclasse di Employee che aggiunge la propriet reports. In questo caso, un'istanza della classe Manager avr tutte e tre le propriet: name, dept, e reports.

JavaScript implementa l'ereditariet permettendo di associare un oggetto prototipo ad ogni funzione costruttore. Quindi, possibile ricreare esattamente l'esempio visto in precedenza, ma usando una terminologia leggermente diversa. Innanzitutto si definisce la funzione costruttore Employee, specificando le propriet name e dept. In seguito, si definisce la funzione costruttore Manager, chiamando il costruttore Employee e specificando la propriet reports. Infine, si assegna a un nuovo oggetto derivato da Employee.prototype come il prototipo per la funzione costruttore Manager. Quando si crea un nuovo Manager, questo eredita le propriet name e dept dall'oggetto Employee.

Aggiungere e rimuovere propriet

Nei linguaggi basati su classi, una classe viene solitamente creata in fase di compilazione mentre le istanze possono essere create in fase di compilazione o in fase di esecuzione. Non possibile cambiare il numero o il tipo di propriet di una classe dopo che questa stata definita. In JavaScript in fase di esecuzione si possono aggiungere o rimuovere propriet a qualunque oggetto. Se si aggiunge una propriet a un oggetto che usato come prototipo per un gruppo di oggetti, anche gli oggetti del gruppo ereditano la nuova propriet.

Riepilogo delle differenze

La tabella seguente fornisce un breve riepilogo di alcune di queste differenze. Il resto di questo capitolo descrive nel detteglio l'uso in JavaScript di costruttori e prototipi per creare una gerarchia di oggetti e lo confronta con la procedura che si userebbe in Java.

Confronto tra il sistema di oggetti basato su classi (Java) e il sistema di oggetti basato su prototipi (JavaScript)
Class-based (Java) Prototype-based (JavaScript)
Classi e istanze sono entit separate. Tutti gli oggetti possono ereditare da un altro oggetto.
Definire una classe con una definizione; istanziare una classe con un metodo costruttore Definire e creare una collezione di oggetti con una funzione costruttore.
Creare un singolo oggetto con l'operatore new. Uguale.
Costruire una gerarchia di oggetti usando la definizione di classe per definire le sottoclassi di classi esistenti. Costruire una gerarchia di oggetti assegnando un oggetto come prototipo associato a una funzione costruttore.
Ereditare le propriet seguendo la catena delle classi. Ereditare le propriet seguendo la catena dei prototipi.
La definizione di classe specifica tutte le propriet di tutte le istanze della classe. Non possibile aggiungere propriet dinamicamente durante l'esecuzione. La funzione costruttore o il prototipo specificano un set iniziale di propriet. possibile aggiungere o rimuovere dinamicamente propriet ai singoli oggetti o all'intero gruppo di oggetti.

L'esempio 'dipendente'

Il resto di questo capitolo usa la gerarchia dei dipendenti mostrata nella figura seguente.

Una semplice gerarchia con gli oggetti seguenti:

  • Employee ha le propriet name (il cui valore di default una stringa vuota) e dept (il cui valore di default "general").
  • Manager basato su Employee. Aggiunge la propriet reports (il cui valore di default un array vuoto, destinato a contenere una serie di oggetti Employee).
  • WorkerBee anch'esso basato su Employee. Aggiunge la propriet projects (il cui valore di default un array vuoto, destinato a contenere una serie di stringhe).
  • SalesPerson basato su WorkerBee. Aggiunge la propriet quota (il cui valore di default 100). Sovrascrive inoltre la propriet dept con il valore "sales", che indica che tutti i venditori si trovano nello stesso dipartimento.
  • Engineer basato su WorkerBee. Aggiunge la propriet machine (il cui valore di default una stringa vuota) e sovrascrive la propriet dept con il valore "engineering".

Creazione della gerarchia

Ci sono diversi modi per definire una funzione costruttore appropriata per implementare la gerarchia dei dipendenti. Il modo scelto per definirli dipende molto da cosa si vuole riuscire a fare nella propria applicazione.

Questa sezione mostra come usare definizioni molto semplici (e relativamente rigide) per dimostrare come far funzionare l'ereditariet. In queste definizioni, quando si crea un oggetto non possibile specificare il valore di nessuna propriet. L'oggetto creato avr semplicemente i valori di default, che potranno essere cambiati in un secondo momento.

In una applicazione reale, probabilmente si vorranno definire dei costruttori che permettono di impostare i valori delle propriet mentre si crea un oggetto (per maggiori informazioni si veda la sezione Costruttori pi flessibili). Per adesso, queste semplici definizioni dimostrano come le propriet vengono ereditate.

Le seguenti definizioni Java e JavaScript di Employee sono simili. L'unica differenza che in Java si ha la necessit di specificare il tipo di ogni propriet (questo dovuto al fatto che Java un linguaggio fortemente tipizzato mentre JavaScript un linguaggio a tipizzazione dinamica.

JavaScript

function Employee() {
  this.name = "";
  this.dept = "general";
}

Java

public class Employee {
   public String name = "";
   public String dept = "general";
}

Le definizioni di Manager e WorkerBee mostrano la differenza nel modo di specificare l'oggetto seguente che si trova pi in alto nella catena di ereditariet. In JavaScript si aggiunge un'istanza prototipo come valore della propriet prototype della funzione costrutore. possibile farlo in qualsiasi momento dopo aver definito il costruttore. In Java, si specifica la superclasse all'interno della definizione della classe. Non possibile cambiare la superclasse all'esterno della definizione.

JavaScript

function Manager() {
  Employee.call(this);
  this.reports = [];
}
Manager.prototype = Object.create(Employee.prototype);

function WorkerBee() {
  Employee.call(this);
  this.projects = [];
}
WorkerBee.prototype = Object.create(Employee.prototype);

Java

public class Manager extends Employee {
   public Employee[] reports = new Employee[0];
}



public class WorkerBee extends Employee {
   public String[] projects = new String[0];
}

Le definizioni di Engineer e SalesPerson creano oggetti che discendono da WorkerBee, e quindi da Employee. Un oggetto di questo tipo ha le propriet di tutti gli oggetti che si trovano sopra di esso nella catena. Inoltre, queste definizioni sovrascrivono il valore ereditato delle propriet dept con nuovi valori specifici per questi oggetti.

JavaScript

function SalesPerson() {
   WorkerBee.call(this);
   this.dept = "sales";
   this.quota = 100;
}
SalesPerson.prototype = Object.create(WorkerBee.prototype);

function Engineer() {
   WorkerBee.call(this);
   this.dept = "engineering";
   this.machine = "";
}
Engineer.prototype = Object.create(WorkerBee.prototype);

Java

public class SalesPerson extends WorkerBee {
   public String dept = "sales";
   public double quota = 100.0;
}


public class Engineer extends WorkerBee {
   public String machine;
   public String dept = "engineering";
   public String machine = "";
}

Usando queste definizioni possibile creare istanze di questi oggetti che ricevono i valori di default delle loro propriet. La figura seguente illustra l'utilizzo di queste definizioni JavaScript per creare nuovi oggetti e mostra i valori delle diverse propriet per questi nuovi oggetti.

Nota: Il termine istanza ha uno specifico significato tecnico nei linguaggi basati su classi. In questi linguaggi, un'istanza una singola istanziazione di una classe ed fondamentalmente differente dala classe. In JavaScript, "istanza" non ha questo significato tecnico perch JavaScript non ha questa differenza tra classi e istanze. Tuttavia, parlando di JavaScript, "istanza" pu essere usato in modo informale per riferirsi a un oggetto creato usando una particolare funzione costruttore. Quindi, in questo esempio, possibile dire che jane un'istanza di Engineer. Allo stesso modo, sebbene i termini genitore (parent), figlio (child), antenato (ancestor) e discendente (descendant) non hanno un significato conenzionale in JavaScript, possono essere usati in modo informale per riferirsi a oggetti che si trovano pi o meno in alto nella catena dei prototipi.

Propriet degli oggetti

Questa sezione spiega come gli oggetti ereditano le propriet da altri oggetti presenti nella catena dei prototipi e cosa succede quando viene aggiunta una propriet in fase di esecuzione.

Ereditare le propriet

Supponiamo di creare un nuovo oggetto WorkerBee chiamato mark con l'istruzione seguente:

var mark = new WorkerBee;

Quando JavaScript vede l'operatore new, crea un nuovo oggeto generico e lo passa come valore della parola chiave this nella funzione costruttore WorkerBee. La funzione costruttore imposta esplicitamente il valore della propriet projects e implicitamente il valore della propriet interna __proto__ uguale al valore WorkerBee.prototype. (Il nome di questa propriet ha due underscores iniziali e due finali). La propriet __proto__ determina la catena di prototipi usata per restituire i valori delle propriet. Una volta che queste propriet sono impostate, JavaScript restituisce il nuovo oggetto e l'istruzione di assegnazione imposta la variabile mark per questo oggetto.

Questo processo non inserisce esplicitamente valori nell'oggetto mark (valori locali) per le propriet che l'oggetto eredita dalla catena dei prototipi. Quando si richiede un valore di una propriet, JavaScript prima controlla se il valore presente nell'oggetto. Se c', viene restituito quel valore. Se il valore non presente a livello locale, JavaScript controlla la catena dei prototipi (sfruttando la propriet __proto__). Se un oggetto nella catena dei prototipi ha un valore per la propriet, viene restituito. Se non viene trovata la propriet, JavaScript risponde che l'oggetto non ha la propriet cercata. In questo modo, l'oggetto mark ha le seguenti propiet con i rispettivi valori:

mark.name = "";
mark.dept = "general";
mark.projects = [];

L'oggetto mark eredita i valori per le propriet name e dept dall'oggetto prototipo presente in mark.__proto__. Il costruttore WorkerBee assegna un valore locale per la propriet projects. L'ereditariet di propriet e valori in JavaScript fnziona in questo modo. Alcune sottigliezze su questo processo sono trattate nella sezione Ereditare le propriet (revisited).

Poich questi costruttori non permettono di fornire valori specifici per una singola istanza, questa informazione generica. I valori delle propriet sono quelli di default condivisi da tutti i nuovi oggetti creati da WorkerBee. oviamente possibile cambiare i valori di ognuna di queste propriet. quindi possibile assegnare informazioni specifice per mark nel modo seguente:

mark.name = "Doe, Mark";
mark.dept = "admin";
mark.projects = ["navigator"];

Aggiungere propriet

In JavaScript, possibile aggiungere propriet a qualsiasi oggetto in fase di esecuzione. Non si costretti ad usare solo le propriet fornite dalla funzione costruttore. Per aggiungere una propriet che sia specifica per un singolo oggetto, si deve assegnare il valore all'oggetto nel modo seguente::

mark.bonus = 3000;

Ora, l'oggetto markha una propriet bonus, ma nessun altro WorkerBee ha questa propriet.

Se si aggiunge una nuova propriet a un oggetto che viene usato come prototipo per una funzione costruttore, la propriet sar aggiunta a tutti gli oggetti che ereditano le propriet dal prototipo. Per esempio, possibile aggiungere la propriet specialty a tutti i dipendenti con l'istruzione seguente:

Employee.prototype.specialty = "none";

Non appena JavaScript esegue questa istruzione, anche l'oggetto mark avr la propriet specialty con il valore "none". La figura seguente mostra cosa succede qunado questa propriet viene aggiunta al prototipo Employee e in seguito la si sovrascrive per il prototipo Engineer.

Queste definizioni JavaScript usano un linguaggio speciale per impostare i valori di default:

this.name = name || "";

In JavaScript l'operatore logico OR (||) valuta il suo primo argomento. Se questa espressione pu essere convertita a true, l'operatore restituisce il primo argomento. Altrimenti l'operatore restituisce il valore del secondo argomento. Quindi, questa linea di codice verifica se name ha un valore utile per la propriet name. Se ce l'ha, imposta this.name a questo valore. Altrimenti, imposta this.name a una stringa vuota. Questo capitolo usa questo linguaggio per brevit, comunque pu disorientare a prima vista.

Nota: Potrebbe non funzionare come atteso se la funzione costruttore chiamata con un argomento che converte a false (come ad esempio 0 (zero) o una stringa vuota ("")). In questo caso verr scelto il valore di default.

Con queste definizioni, quando si crea un'istanza di un oggetto, possibile specificare i valori per le propriet definite localmente. Per creare un nuovo Engineer possibile utilizzare l'espressione seguente:

var jane = new Engineer("belau");

Le propriet di Jane sono ora:

jane.name == "";
jane.dept == "engineering";
jane.projects == [];
jane.machine == "belau"

da notare che con queste definizioni non possibile specificare una valore iniziale per le propriet ereditate, come ad esempio name. Se si desidera specificare un valore iniziale per una propriet ereditata, in JavaScript necessario aggiungere ulteriore codice alla funzione costruttore.

Finora, la funzione costruttore ha creato un oggetto generico e poi ha specificato propriet locali e valori per il nuovo oggetto. possibile fare in modo che il costruttore aggiunga pi propriet chiamando direttamente la funzione costruttore per un oggetto che si trova a un livello pi alto nella catena dei prototipi. La figura seguente mostra queste nuove definizioni.


Specificare le propriet in un costruttore, modo 2

Guardiamo nel dettaglio una di queste definizioni. Ecco la nuova definizione per il costruttore Engineer:

function Engineer (name, projs, mach) {
  this.base = WorkerBee;
  this.base(name, "engineering", projs);
  this.machine = mach || "";
}

Supponiamo di creare un nuovo oggetto Engineer nel modo seguente:

var jane = new Engineer("Doe, Jane", ["navigator", "javascript"], "belau");

JavaScript segue questa procedura:

  1. L'operatore new crea un oggetto generico e imposta il valore Engineer.prototype per la sua propriet __proto__.
  2. L'operatore new passa il nuovo oggetto come valore della parola chiave this nella funzione costruttore Engineer.
  3. Il costruttore crea una nuova propriet per l'oggetto chiamata base e assegna il valore del costruttore WorkerBee alla propriet base. Questo rende il costruttore WorkerBee un metode dell'oggetto Engineer. Il nome della propriet base non peculiare. possibile usare qualsiasi nome per la propriet; base semplicemente evocativo dello scopo.
  4. Il costruttore chiama il metodo base, passando come suoi argomenti due degli argomenti passati al costruttore ("Doe, Jane" e ["navigator", "javascript"]) e anche la stringa "engineering". Usando esplicitamente "engineering" nel costruttore indica che tutti gli oggetti Engineer hanno lo stesso valore per la propriet dept ereditata, e questo valore sovrascrive il valore ereditato dal costruttore Employee.
  5. Poich base un metodo di Engineer, all'interno della chiamata di base, JavaScript aggiunge la parola chiave this all'oggetto creato al passaggio 1. In questo modo, la funzione WorkerBee a sua volta passa gli argomenti "Doe, Jane" e "engineering" alla funzione costruttore Employee. Dopo l'esecuzione della funzione costruttore Employee, la funzione WorkerBee utilizza l'argomento rimanente per impostare la propriet projects.
  6. Dopo l'esecuzione del metodo base, il costruttore Engineer inizializza la propriet machine dell'oggetto al valore "belau".
  7. Dopo l'esecuzione del costruttore, JavaScript assegna il nuovo oggetto alla variabile jane.

Avendo chiamato il costruttore WorkerBee dall'interno del costruttore Engineer, si potrebbe pensare di aver impostato in modo corretto l'ereditariet per gli oggetti Engineer. Questo non il caso. Chiamare il costruttore WorkerBee assicura che un oggetto Engineer venga creato con le propriet specificate in tutte le funzioni costruttore che sono chiamate. Per, se in un secondo momento vengono aggiunte propriet ai prototipi Employee o WorkerBee, queste propriet non saranno ereditate dall'oggetto Engineer. Per esempio, se si considera il codice seguente:

function Engineer (name, projs, mach) {
  this.base = WorkerBee;
  this.base(name, "engineering", projs);
  this.machine = mach || "";
}
var jane = new Engineer("Doe, Jane", ["navigator", "javascript"], "belau");
Employee.prototype.specialty = "none";

L'oggetto jane non eredita la propriet specialty. comunque necessario impostare esplicitamente il prototipo per garantire l'ereditariet dinamica. Se si considera invece l'esempio seguente:

function Engineer (name, projs, mach) {
  this.base = WorkerBee;
  this.base(name, "engineering", projs);
  this.machine = mach || "";
}
Engineer.prototype = new WorkerBee;
var jane = new Engineer("Doe, Jane", ["navigator", "javascript"], "belau");
Employee.prototype.specialty = "none";

Adesso il valore per l'oggetto jane della propriet specialty "none".

Un altro modo per ereditare le propriet l'utilizzo dei metodi call() e apply(). Gli esempi sottostanti si equivalgono:

function Engineer (name, projs, mach) {
  this.base = WorkerBee;
  this.base(name, "engineering", projs);
  this.machine = mach || "";
}

function Engineer (name, projs, mach) {
  WorkerBee.call(this, name, "engineering", projs);
  this.machine = mach || "";
}

L'utilizzo del metodo call() costituisce un'implementazione pi pulita poich la propriet base non pi necessaria.

Ereditare le propriet (revisited)

Le sezioni precedenti descrivono come i costruttori e i prototipi consentono di avere gerarchia ed ereditariet in JavaScript. Questa sezione espone alcune sottiglienzze che non erano necessariamente evidenti nelle discussioni precedenti.

Valori locali vs. valori ereditati

Quando si accede a una propriet di un oggetto, JavaScript esegue i seguenti passaggi, come descritto in precedenza in questo capitolo:

  1. Verifica se il valore presente a livello locale. Se c', restituisce quel valore.
  2. Se non presente, verifica la catena dei prototipi (usando la propriet __proto__).
  3. Se un oggetto nella catena dei prototipi ha un valore per la propriet specificata, restituisce quel valore.
  4. Se la propriet non viene trovata, l'oggetto non ha la propriet.

Il risultato di questo processo dipende da come sono stati definiti gli elementi. L'esempio iniziale aveva queste definizioni:

function Employee () {
  this.name = "";
  this.dept = "general";
}

function WorkerBee () {
  this.projects = [];
}
WorkerBee.prototype = new Employee;

Con queste definizioni, si supponga di creare amy come un'istanza di WorkerBee con la seguente istruzione:

var amy = new WorkerBee;

L'oggetto amy ha una propriet locale, projects. I valori per le propriet name e dept non sono specifici per amy e quindi derivano dalla propriet __proto__ dell'oggetto amy. Quindi, amy ha i seguenti valori:

amy.name == "";
amy.dept == "general";
amy.projects == [];

Ora si supponga di cambiare il valore della propriet name nel prototipo associato con Employee:

Employee.prototype.name = "Unknown"

Ci si potrebbe aspettare che il nuovo valore si propaghi a tutte le istanze di Employee. Invece, non lo fa.

Quando si crea qualsiasi istanza dell'oggetto Employee, questa istanza riceve un valore locale per la propriet name (la stringa vuota). Questo significa che quando si imposta il prototipo WorkerBee creando un nuovo oggetto Employee, WorkerBee.prototype avr un valore locale per la propriet name. Quindi, quando JavaScript legge la propriet name dell'oggetto amy (istanza di WorkerBee), trova in WorkerBee.prototype il valore locale di questa propriet. Pertanto non continua a cercare nella catena salendo a Employee.prototype.

Se si vuole cambiare il valore di una propriet di un oggetto in fase di esecuzione e si vuole che il nuovo valore venga ereditato da tutti i discendenti dell'oggetto, non possibile definire la propriet all'interno della funzione costruttore dell'oggetto. Invece, si aggiunge la propriet al prototipo associato al costruttore. Per esempio, assumiamo di cambiare il codice precedente con quello che segue:

function Employee () {
  this.dept = "general";    // Note that this.name (a local variable) does not appear here
}
Employee.prototype.name = "";    // A single copy

function WorkerBee () {
  this.projects = [];
}
WorkerBee.prototype = new Employee;

var amy = new WorkerBee;

Employee.prototype.name = "Unknown";

In questo caso, il valore della propriet name di amy diventa "Unknown".

Come mostra questo esempio, se si vogliono avere i valori di default per le propriet dell'oggetto e si vuole avere la possibilit di cambiare questi valori in fase di esecuzione, si devono impostare le propriet nel prototipo del costruttore, e non direttamente nella funzione costruttore.

Determinazione delle relazioni tra istanze

In JavaScript la ricerca delle propriet (property lookup) controlla prima tra le propriet specifiche dell'oggetto e, se il nome della propriet non viene trovato, controlla la propriet speciale __proto__. Il processo, chiamato "ricerca nella catena dei prototipi" (lookup in the prototype chain), continua ricorsivamente .

Quando un oggetto viene costruito, la propriet speciale __proto__ viene impostata al valore della propriet prototype del costruttore. L'espressione new Foo() crea un oggetto con __proto__ == Foo.prototype. Di conseguenza, le modifiche alle propriet di Foo.prototype alterano la ricerca delle propriet per tutti gli oggetti che sono stati creati con new Foo().

Ogni oggetto ha una propriet __proto__ (ad eccezione di Object); ogni funzione ha una propriet prototype. Quindi gli oggetti possono essere correlati ad altri oggetti attraverso una 'ereditariet prototipale' (prototypal inheritance). possibile verificare l'ereditariet confrontando la propriet __proto__ di un oggetto con l'oggetto prototype di una funzione. JavaScript fornisce una scorciatoia: l'operatore instanceof confronta un oggetto con una funzione e restituisce true se l'oggetto eredita dal prototipo della funzione. Per esempio,

var f = new Foo();
var isTrue = (f instanceof Foo);

Per un esempio pi dettagliato, supponiamo di avere lo stesso gruppo di definizioni visto nella sezione For a more detailed example, suppose you have the same set of definitions shown in Ereditare le propriet. Creiamo un oggetto Engineer nel modo seguente:

var chris = new Engineer("Pigman, Chris", ["jsd"], "fiji");

Per questo oggetto, tutti gli enunciati seguenti sono veri:

chris.__proto__ == Engineer.prototype;
chris.__proto__.__proto__ == WorkerBee.prototype;
chris.__proto__.__proto__.__proto__ == Employee.prototype;
chris.__proto__.__proto__.__proto__.__proto__ == Object.prototype;
chris.__proto__.__proto__.__proto__.__proto__.__proto__ == null;

Dato ci, possibile scrivere una funzione instanceOf come segue:

function instanceOf(object, constructor) {
   object = object.__proto__;
   while (object != null) {
      if (object == constructor.prototype)
         return true;
      if (typeof object == 'xml') {
        return constructor.prototype == XML.prototype;
      }
      object = object.__proto__;
   }
   return false;
}

Nota: L'implementazione vista sopra verifica se l'oggetto di tipo "xml" per ovviare a una stranezza nel modo in cui gli oggetti XML sono rappresentati nelle versioni recenti di JavaScript. Per i dettagli essenziali si veda il bug634150.

Se si utilizza la funzione instanceOf definita in precedenza, queste espressioni sono vere:

instanceOf (chris, Engineer)
instanceOf (chris, WorkerBee)
instanceOf (chris, Employee)
instanceOf (chris, Object)

Ma l'espressione seguente falsa:

instanceOf (chris, SalesPerson)

Informazioni globali nei costruttori

Quando vengono creati dei costruttori, necessario essere scrupolosi se si impostano informazioni globali all'interno del costruttore. Per esempio, se si vuole che un ID univoco venga assegnato automaticamente a ogni nuovo Employee si potrebbe usare la definizione seguente:

var idCounter = 1;

function Employee (name, dept) {
   this.name = name || "";
   this.dept = dept || "general";
   this.id = idCounter++;
}

Con questa definizione, quando viene creato un nuovo Employee, il costruttore assegna l'ID seguente e incrementa il contatore globale (idCounter). Cos, se l'istruzione successiva la seguente, l'ID di victoria sar 1, l'ID di harry sar 2:

var victoria = new Employee("Pigbert, Victoria", "pubs")
var harry = new Employee("Tschopik, Harry", "sales")

Questa a prima vista potrebbe sembrare la procedura corretta. Tuttavia, il contatore globale idCounter viene incrementato ogni volta che viene creato un oggetto Employee, per qualsiasi scopo. Se viene creata l'intera gerarchia di oggetti Employee mostrata in questo capitolo, il costruttore Employee viene chiamato ogni volta che si definisce un prototipo. Supponiamo di avere il codice seguente:

var idCounter = 1;

function Employee (name, dept) {
   this.name = name || "";
   this.dept = dept || "general";
   this.id = idCounter++;
}

function Manager (name, dept, reports) {...}
Manager.prototype = new Employee;

function WorkerBee (name, dept, projs) {...}
WorkerBee.prototype = new Employee;

function Engineer (name, projs, mach) {...}
Engineer.prototype = new WorkerBee;

function SalesPerson (name, projs, quota) {...}
SalesPerson.prototype = new WorkerBee;

var mac = new Engineer("Wood, Mac");

Si assuma inoltre che le definizioni omesse abbiano la propriet base e chiamino il costruttore che si trova al livello superiore nella catena dei prototipi. In questo caso, nel momento in cui viene creato l'oggetto mac, il valore di mac.id sar 5.

A seconda dell'applicazione, pu essere pi o meno importante che il valore del contatore sia stato incrementato queste volte aggiuntive. Se interessa il valore esatto di questo contatore, una soluzione possibile pu prevedere l'uso del costruttore seguente al posto di quello visto in precedenza:

function Employee (name, dept) {
   this.name = name || "";
   this.dept = dept || "general";
   if (name)
      this.id = idCounter++;
}

Quando viene creata un'istanza di Employee che deve essere usata come prototipo, non vengono forniti argomenti al costruttore. Usando questa definizione del costruttore, quando non vengono inseriti argomenti, il costruttore non assegna un valore all'ID e non aggiorna il contatore. Quindi, affinch a un oggetto Employee venga assegnato un ID, necesario specificare un nome per il dipendente. In questo esempio, l'ID di mac sar 1.

JavaScript non supporta l'ereditariet multipla

Alcuni linguaggi ad oggetti ammettono l'ereditariet multipla. Ossia, un oggetto pu ereditare propriet e valori da oggetti genitori non correlati. Javascript non supporta l'ereditariet multipla.

L'eredit dei valori delle propriet si ha in fase di esecuzione quando JavaScript cerca attraverso la catena dei prototipo di un oggetto per trovare un valore. Poich un oggetto ha un unico prototipo associato, JavaScript non pu ereditare dinamicamente da pi di una catena di prototipi.

In JavaScript, possibile che una funzione costruttore chiami al suo interno diverse funzioni costruttore. Questo d l'illusione dell'ereditariet multipla. Per esempio consideriamo le istruzioni seguenti:

function Hobbyist (hobby) {
   this.hobby = hobby || "scuba";
}

function Engineer (name, projs, mach, hobby) {
   this.base1 = WorkerBee;
   this.base1(name, "engineering", projs);
   this.base2 = Hobbyist;
   this.base2(hobby);
   this.machine = mach || "";
}
Engineer.prototype = new WorkerBee;

var dennis = new Engineer("Doe, Dennis", ["collabra"], "hugo")

Assumiamo inoltre che la definizione di WorkerBee sia quella usata in precedenza in questo capitolo. In questo caso, l'oggetto dennis avr queste propriet:

dennis.name == "Doe, Dennis"
dennis.dept == "engineering"
dennis.projects == ["collabra"]
dennis.machine == "hugo"
dennis.hobby == "scuba"

Quindi dennis riceve la propriet hobby dal costruttore Hobbyist. Per, se in seguito si aggiunge una propriet al prototipo del costruttore Hobbyist:

Hobbyist.prototype.equipment = ["mask", "fins", "regulator", "bcd"]

L'oggetto dennis non erediter questa nuova propriet.
