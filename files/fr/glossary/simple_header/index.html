Un en-tête simple (ou en-tête de requête sécurisé CORS) est l'un des en-têtes HTTP suivants :

  • Accept,
  • Accept-Language,
  • Content-Language,
  • Content-Type avec un type MIME de sa valeur analysée (ignorant les paramètres) et l'un des application/x-www-form-urlencoded, multipart/form-data ou text/plain.

Ou l'un de ces en-têtes d'indication de client :

  • DPR
  • Downlink
  • Save-Data
  • Viewport-Width
  • Width

Lorsqu'elles ne contiennent que de simples en-têtes, les requêtes sont réputées simples et n'ont pas besoin d'envoyer une requête de pré-vérification dans le contexte de CORS.
